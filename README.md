## How I like to build

- Start from a real user problem, not a buzzword.
- Keep AI useful, explainable, and honest about its limits.
- Prefer local-first and privacy-aware systems when possible.
- Turn rough ideas into working prototypes, then harden what proves useful.
- Make the product feel clear, alive, and worth coming back to.
